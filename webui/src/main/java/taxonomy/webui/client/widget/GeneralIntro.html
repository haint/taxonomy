TAXONOMY DATABASE SYSTEM

Besides the main database, two optional ones are for taxonomic families and genera, which store additional information on families and genera, respectively

It is important that you enter Family names in the main database in corresponding with name in the Family database. This is to ensure that the two databases are linked with each other properly. The same is true for Genus names in the main database and in the Genus database.

There is one database for Keywords, and another for Indexing terms. You should maintain these databases to be consistent with the main ones.

The system is designed based on the one-entry one-exit basics. This means that you should exit from the system by the first menu so that file are closed properly. Falling to do so may corrupt your databases.

Deprecated

INDEXING is normally not neccessary. It is required only when data indecies are corrupted due to some reason. You may try it when having problems.

DATA VALIDATION is to maintain data integrity. For this reason, it is run only by personnel in charge.

Although this is rare, a problem may cause the program to stop and ask if you want to cancel, ignore or suspend. Try to remember what you just did, note down the message, then type C to CANCEL. Then enter the command QUIT to exit from the program. Stop using the program until the problem has been fixed.
It is very difficult to rebuild a lost database. So, remember: BACKUP and BACKUP !
